# Learning Journal Templates

This directory contains templates for structured learning capture.
Use them for specific types of learnings that the CLI capture tool
doesn't cover directly.

## Available Templates

### `bug-analysis.md`
Use when you've debugged a complex bug. Captures the full cycle:
symptoms → investigation → root cause → fix → prevention.

### `decision-retro.md`
Use when a previous decision turned out wrong. Captures why,
what signals were missed, and how to decide better next time.

### `user-feedback.md`
Use when you get feedback from someone trying the product.
Structured to separate observation from interpretation.

### `architecture-insight.md`
Use when you discover something about the system's behavior
that wasn't obvious from the diagrams. The gap between
"how it's designed" and "how it actually works."

---

## How to use templates

```bash
cp docs/journal/templates/bug-analysis.md docs/journal/2026-06-XX-bug-name.md
# Edit the file directly
```
